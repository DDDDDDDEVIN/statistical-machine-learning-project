# Machine-Generated Text Detection with Domain-Aware Modeling

This project investigates the task of distinguishing human-written text from machine-generated content under realistic domain shift and class imbalance conditions. We explore both domain-specific and domain-unified modeling frameworks using a BERT-based encoder and evaluate the effectiveness of MMD-based alignment and semi-supervised learning.

## Repository Structure

```
data/
  raw/         input data (domain1_train_data.json, domain2_train_data.json, test_data.json, ...)
  landing/     intermediate data merged from raw (gitignored, regenerated by the notebooks)
  curated/     cleaned, unified-schema training data (gitignored, regenerated by the notebooks)
notebooks/     experiment notebooks (see below)
src/           shared model/data-pipeline code imported by the notebooks
checkpoints/   trained model weights, one subfolder per notebook (gitignored)
results/       prediction CSVs written by the notebooks (gitignored)
plots/         confusion-matrix figures written by the notebooks (gitignored)
```

### Notebooks

| Notebook | Description |
|---|---|
| `bert_baseline.ipynb` | BERT-based classifier, trained directly on domain1+domain2 (baseline). |
| `mlp_baseline.ipynb` | MLP classifier, same setup as above (baseline). |
| `specific_lstm.ipynb` | Domain classifier + domain-specific LSTM classifiers. |
| `specific_mlp.ipynb` | Domain classifier + domain-specific MLP classifiers. |
| `MAIN_unified_SMOTE_semi.ipynb` | Unified domain-adapted model with semi-supervised pseudo-labeling and SMOTE oversampling. **Final model.** |
| `unified_mixup_semi.ipynb` | Same semi-supervised pipeline as above, using Mixup instead of SMOTE. |

`bert_baseline`/`mlp_baseline` share their data pipeline and training driver via `src/baselines.py`; `MAIN_unified_SMOTE_semi`/`unified_mixup_semi` share theirs via `src/semi.py` (each notebook supplies only its own phase-2 fine-tuning step — SMOTE vs. Mixup). `specific_lstm`/`specific_mlp` are self-contained, since most of their code is exploratory and doesn't factor out cleanly.

## Setup

Requires Python >=3.8.

```bash
pip install -r requirements.txt
```

A GPU is recommended (CUDA or Colab) but not required — the notebooks fall back to CPU automatically.

## Data

Place the raw input files in `data/raw/`:
- `domain1_train_data.json`, `domain2_train_data.json` — training data for the two domains
- `test_data.json` — held-out evaluation data

`data/landing/` and `data/curated/` are produced by `MAIN_unified_SMOTE_semi.ipynb` (merged and cleaned versions of the raw data) and are not tracked in git — run the notebook to regenerate them.

## Running Experiments

Notebooks assume they are run from inside `notebooks/` (paths like `../data/raw/...` are relative to that). Run in this order to reproduce results:

1. Baselines: `bert_baseline.ipynb`, `mlp_baseline.ipynb`
2. Domain-specific: `specific_lstm.ipynb`, `specific_mlp.ipynb`
3. Domain-unified (semi-supervised): `MAIN_unified_SMOTE_semi.ipynb`, `unified_mixup_semi.ipynb`

Each notebook trains its model, checkpoints the best weights to `checkpoints/<name>/`, and (where the `test()`/`main.test()` call is enabled) writes predictions to `results/`.

## Notes

- All models are trained from scratch, without pretrained weights.
- Evaluation is based on accuracy and F1-score.
- `test_data.json` contains a small number of empty-text rows; `bert_baseline`, `mlp_baseline`, and `unified_mixup_semi` currently leave their `test()` call disabled for this reason (see the notebooks for details).
