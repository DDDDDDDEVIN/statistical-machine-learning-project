# Robust Machine Text Detection under Domain Shift via MMD-Guided Semi-Supervised Learning

An NLP project that detects machine-generated text under realistic domain shift
and severe class imbalance, comparing a domain-specific pipeline against a
unified, domain-agnostic model trained with MMD representation alignment and
semi-supervised pseudo-labeling.

> **Academic team project:** This repository is a portfolio edition of a group
> project (COMP90051 Statistical Machine Learning, Project 2, Group 60)
> completed at the University of Melbourne in May 2025. It preserves the
> original analysis while presenting the work in a cleaner, more reproducible
> format. Results reflect the original coursework dataset and Kaggle-style
> competition setup; they are not a benchmark for a production text-detection
> system.

**Portfolio owner and maintainer:** Angqi Meng

## Project overview

Large language models make it increasingly hard to tell human-written text
from machine-generated text, and detectors that work well in one setting often
fail to generalize to another. The project studies this under two realistic
difficulties at once:

- **Domain shift** — the training data spans two domains with different
  writing styles, and domain labels are not available at test time.
- **Class imbalance** — Domain 1 is balanced (500 human / 500 machine), while
  Domain 2 is heavily skewed (4,750 machine / 250 human) across 5,000 training
  samples in total.

We compare two complementary strategies for this: a **domain-specific**
pipeline that predicts the domain first and routes to a domain-conditioned
classifier, and a **domain-unified** model that instead learns
domain-invariant, label-discriminative representations directly, using MMD
(Maximum Mean Discrepancy) alignment between same-label embeddings across
domains, plus semi-supervised pseudo-labeling to exploit high-confidence
validation predictions as extra training signal.

The full write-up is in [`docs/report.pdf`](docs/report.pdf).

## Highlights

Results below are from the team's final report (`docs/report.pdf`, Tables 1–2),
evaluated on the held-out validation set with accuracy and macro-averaged F1:

| Model | Accuracy | F1 | Notebook |
| --- | --- | --- | --- |
| MLP baseline | 0.8423 | 0.7756 | `mlp_baseline.ipynb` |
| BERT baseline | 0.9267 | 0.9060 | `bert_baseline.ipynb` |
| Domain-specific (MLP heads) | 0.9350 | 0.8259 | `specific_mlp.ipynb` |
| Domain-specific (LSTM heads) | 0.8900 | 0.8059 | `specific_lstm.ipynb` |
| Domain-unified, supervised only | 0.9352 | 0.9122 | phase 1 of `MAIN_unified_SMOTE_semi.ipynb` |
| **Domain-unified, semi-supervised (final)** | **0.9817** | **0.9583** | `MAIN_unified_SMOTE_semi.ipynb` |

Ablations on the final model (Table 2 of the report):

- Removing the MMD alignment loss drops accuracy to 0.9336 (F1 0.9295) —
  cross-domain representation alignment is doing real work, not just the
  semi-supervised step.
- Swapping SMOTE for Mixup as the resampling strategy (`unified_mixup_semi.ipynb`)
  gives 0.9773 accuracy / 0.9482 F1 — close to SMOTE, suggesting the semi-supervised
  signal matters more than the specific resampling technique.

### Notebooks

| Notebook | Description |
|---|---|
| `bert_baseline.ipynb` | BERT-based classifier trained directly on domain1+domain2 (baseline). |
| `mlp_baseline.ipynb` | MLP classifier, same setup as above (baseline). |
| `specific_lstm.ipynb` | Domain classifier + domain-specific LSTM classifiers. |
| `specific_mlp.ipynb` | Domain classifier + domain-specific MLP classifiers. |
| `MAIN_unified_SMOTE_semi.ipynb` | Domain-unified model: MMD alignment + SMOTE oversampling + semi-supervised pseudo-labeling. **Final model.** |
| `unified_mixup_semi.ipynb` | Same domain-unified pipeline, using Mixup instead of SMOTE for phase-2 resampling. |

`bert_baseline`/`mlp_baseline` share their data pipeline and training driver via
`src/baselines.py`; `MAIN_unified_SMOTE_semi`/`unified_mixup_semi` share theirs
via `src/semi.py` (each notebook supplies only its own phase-2 fine-tuning step
— SMOTE vs. Mixup). `specific_lstm`/`specific_mlp` are self-contained, since
most of their code is exploratory and doesn't factor out cleanly.

## Reproducing the analysis

Requires Python >=3.8. A GPU is recommended (the original experiments used an
RTX 4090) but not required — the notebooks fall back to CPU automatically.

```bash
python -m venv .venv
source .venv/bin/activate      # or .venv\Scripts\activate on Windows
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
jupyter lab
```

Place the raw input files in `data/raw/`: `domain1_train_data.json`,
`domain2_train_data.json`, and `test_data.json`. Notebooks assume they run
from inside `notebooks/` (paths like `../data/raw/...` are relative to that).

Run in this order to reproduce results:

1. Baselines: `bert_baseline.ipynb`, `mlp_baseline.ipynb`
2. Domain-specific: `specific_lstm.ipynb`, `specific_mlp.ipynb`
3. Domain-unified: `MAIN_unified_SMOTE_semi.ipynb` (also produces
   `data/landing/` and `data/curated/`), then `unified_mixup_semi.ipynb`

Each notebook checkpoints its best weights to `checkpoints/<name>/` and, where
the `test()` call is enabled, writes predictions to `results/`.

## Methods

- **Preprocessing:** TF-IDF weighting and n-gram tokenization, padded to a
  fixed length of 256; empty-text rows are dropped in the curated data layer.
- **Domain-specific modeling:** a domain classifier routes each input to a
  domain-conditioned classifier (MLP or LSTM head), so each head only has to
  learn one domain's label semantics.
- **Domain-unified modeling:** a single randomly-initialized BERT encoder
  (no pretraining) with a residual feedforward decoder head, trained with
  cross-entropy plus an MMD loss that aligns same-label embeddings across
  domains — encouraging domain-invariant, label-discriminative features. MMD
  is activated after a warm-up period so the encoder first stabilizes under
  supervised loss alone.
- **Class-imbalance handling:** SMOTE and Mixup resampling on the encoder's
  `[CLS]` embeddings, applied during a frozen-encoder phase-2 fine-tuning step.
- **Semi-supervised learning:** validation predictions with confidence >=0.99
  are added as pseudo-labeled training data before phase-2 fine-tuning.
- **Evaluation:** accuracy and macro-averaged F1 on a held-out validation set.

## Limitations and responsible use

- The BERT encoder is trained from scratch (no pretrained weights), so results
  are specific to this dataset and are not directly comparable to pretrained
  language-model detectors.
- Domain labels are unavailable at test time by design; the domain-specific
  pipeline's accuracy depends on the upstream domain classifier being correct.
- Semi-supervised pseudo-labels are drawn from the validation set itself,
  which is appropriate for this coursework setting but would need a genuinely
  held-out unlabeled pool in a production system.
- `test_data.json` contains a small number of empty-text rows; several
  notebooks currently leave their final `test()` call disabled as a result
  (see the notebooks for details).
- This is a historical academic project on a fixed, relatively small dataset
  (5,000 training samples); it should not be treated as a general-purpose or
  currently competitive machine-text detector.
